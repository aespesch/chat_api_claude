Seu objetivo é criar um aplicativo python para streamlit capaz de coletar dados de reservas de hotel.

O aplicativo será executado em http://turma90_hotel.streamlit.io e o código deve ser criado pela modificação dos arquivos app.py e config.py em anexo, os quais são usados para pagar um almoço comemorativo. Os scripts fornecidos são usados em http://turma90.streamlit.io.

De forma análoga ao aplicativo em anexo, o aplicativo que você vai criar começa confirmando que o usuário tem seu nome cadastrado no arquivo participants.csv o qual possui os campos e dados exemplificados abaixo:
full_name,id,participants
José da Silva Sauro,0,0
Abílio Neves Garcia,1,0
Adilson Roberto De Marchi Toledo,2,0
Adriano de Conti Bernardi,3,0
Afonso Carlos Azevedo Pereira,4,0
Alan do Amaral Fernandes,5,1
Alexandre Arthur Massena Javoski,6,2

O usuário digita o próprio nome completo e clica no botão verificar. Se o nome digitado estiver no campo full_name, o programa prossegue. Caso contrário, informa o usuário que o nome não está cadastrado e volta à página inicial de modo que o usuário possa fazer uma nova tentativa.

A comparação entre o que o usuário digitou e o que está em full_name de participants.csv deve levar em conta:
1. Uso de acentos: antônio e antonio são iguais.
2. Uso de maiúsculas e minúsculas: Antonio e antonio são iguais.
3. Uso de formas abreviadas: Antonio Espeschit e Antonio Magno Lima Espeschit são iguais.
4. Pequenos erros de digitação: Antnio e Antonio são iguais. Antpnio e Antonio são iguais.

Após ter verificado o nome, vai para uma interface de seleção de quartos.

Serão apresentados vários tipos de quartos, cada um com preço diferente, e o usuário poderá escolher de 0 a N quartos.

Nesta etapa, se o campo participants no arquivo participants.csv for maior que zero, a interface deve exibir:
"Participantes do almoço na Churrascaria Boigalê:" e, em seguinda, o valor de participants obtido em participants.csv.

A interface deve mostrar uma lista de quartos tal como descrito no arquivo rooms.csv que possui os campos e dados exemplificados abaixo:
id,hotel,type,value
1,ibis,sgl,500
2,ibis,dbl,610
3,mercure,sgl,705
4,mercure,dbl,905

Deve ter um texto explicando:
DBL: Quarto para 2 pessoas com 1 cama de casal ou 2 de solteiro
SGN: Quarto para 1 pessoa com 1 cama de casal ou 2 de solteiro
Check-In: Sexta-Feira 28NOV2025 14:00
Check-Out: Sábado 30NOV2025 14:00
Café da manhã incluído.
Estacionamento não incluído.
Se precisar de algo diferente, use o campo observações.

O usuário pode definir a quantidade de cada quarto sendo que o valor mínimo a selecionar é 1 quarto. Só habilita o botão prosseguir depois que escolheu pelo menos um quarto.

Quando clica no botão prosseguir, mostra um campo observações. Permite que o usuário escreva no campo observações (opcional) e, em seguida, clique no botão prosseguir. Exemplos de observaçoes: cama extra, pessoa com problemas de locomoção.

Calcula o valor total a pagar. Multiplica a quantidade de quartos selecionados pelo valor (value) de cada um e depois soma tudo.

Grava os dados coletados em RESERVAS.CSV.

Para o valor a pagar, deve ser gerado um código QR Code e uma string de pagamento PIX similar ao que já é feito no APP.PY anexo. Porém, ao invés de ter, no comentário do PIX, uma string "ID<id_da_pessoa>ID", colocar "HH<id_da_pessoa>HH". Isso vai permitir identificar a pessoa que pagou, o valor pago e se a pessoa pagou almoço ou hotel apenas olhando o comprovante do PIX na minha conta.

Depois que gerar o QR Code, mostra instruções de pagamento tal como já é feito no aplicativo APP.PY anexo.
